## Architecture

**La Megaporra** is a World Cup 2026 football prediction-pool leaderboard. ~150 participants are scored across 35 rounds ("jornadas"). Data is fetched live from a public Google Sheets CSV on every page load.

### Single-component design

The entire app lives in **`src/App.vue`**. There are no sub-components. All state, computed rankings, and both views (leaderboard + participant dossier) are in this one file. All styles are inline in the template — there is no scoped `<style>` block and no CSS utility library.

### Router is headless

`src/router/index.js` uses `createWebHashHistory` (`#/` URLs). Vue Router is used **only for URL management** — there is no `<router-view>` anywhere. App.vue reads `useRoute()` / `useRouter()` directly:

- `#/` → `view = 'clasificacion'`
- `#/participant/:id` → `view = 'buscate'`, `selectedName = decodeURIComponent(id)`

Route components in the router config are `{ render: () => null }` stubs, because App.vue handles all rendering itself.

### Data flow

CSV source URL (hardcoded in `src/utils/csvParser.js`):
```
https://docs.google.com/spreadsheets/d/1mBLMjjyqSm5E7pnq2G4hJVSJuJJDl3lYrJfuC43diHo/export?format=csv&gid=0
```

```
Google Sheets CSV
  → src/utils/csvParser.js (fetchData)
      returns { players, jornadas, lastPlayed }
  → App.vue (load())
      players: [{ name, scores: (number|null)[], total }]
      jornadas: [{ n, phase, date, dateTag, cols[] }]
```

**CSV structure (3 header rows):**
- Row 0: `JORNADA N` labels — some jornadas span multiple sub-columns (fill-forward logic groups them)
- Row 1: phase names (`FASE GRUPOS`, `OCTAVOS`, etc.) — `EXTRES` marks the stop boundary
- Row 2: dates + `TOTAL` marker in the last data column
- Rows 3+: player name, per-column scores, total

Sub-column scores (e.g. jornada 14 has 3 CSV columns) are **summed** into a single `scores[k]` entry. `null` means not yet played.

### Ranking logic (in App.vue)

`computeRanking(u)` computes competition ranking (ties share position: 1,1,3…) using cumulative scores up to jornada `u`. The jornada rail slider changes `jornadaNum`, which re-ranks the entire leaderboard. Evolution arrows compare ranking at `u` vs `u-1`.

### Verdicts and micro-tags

All copy in the participant dossier (Búscate view) is generated by the `verdict(pos, n)` function — fully deterministic, offline, no AI. Thresholds are relative to `n` so they keep working as the tournament progresses. Micro-tags in the ranking list follow the same pattern, triggered only near frontier positions.

### Sparkline tooltip

The sparkline bar group in the ranking table uses a `<Teleport to="body">` tooltip (not a CSS `::after`) to escape the `overflow:hidden` on the ranking container. State: `sparkTip = ref({ show, x, y, text })`. Handlers: `onSparkEnter(evt, text)`, `onSparkMove(evt)`, `onSparkLeave()`. The tooltip text is pre-built per row as `sparkTooltip` (e.g. `"J1: 7 pts · J2: 9 pts"`).

### Jornada rail scroll

`scrollRail(instant)` fires after data loads and on every `jornadaNum` change. It right-aligns the active chip (`.rail-active`) inside the `railEl` container using `scrollLeft` (instant on mount) or `scrollTo` with smooth (user interaction).

**Critical**: `railEl` is inside `v-else-if="view === 'clasificacion'"` which only renders when `loading === false`. Always set `loading.value = false` **before** calling `scrollRail` — otherwise `railEl.value` is null and the scroll is a silent no-op.

### Búscate profile — neighbor sparklines

The "QUIÉN TE RODEA EN EL ESCALAFÓN" table shows 7 rows (3 above, you, 3 below). Each row includes a compact per-round sparkline (4×22px bars, same gold gradient as the leaderboard) built from `x.scores.slice(0, lastPlayed)` inside the `mePlayer` computed. The bars sit between the player name and their points total.

### Design system

The design spec lives in `src/assets/DESIGN_SPEC.md` and the reference prototype in `src/assets/La Megaporra.dc.html`. These are the sources of truth for any visual change.

- **Background**: `#0e1016`; panels `#161b27` / `#1c2231`
- **Gold**: `#e7b656` (border), `#f7d684` (bright), `#a9843c` (dim)
- **Green** (Becario / up arrows): `#46b884`
- **Red** (Miseria / down arrows): `#dd5d42`
- **Fonts**: Space Mono (marks/numbers), Archivo (names/UI), Spectral italic (Pako voice/chronicle)
- **Mobile breakpoint**: `window.innerWidth < 760` → hides sparklines, compacts layout

All copy is in Castilian Spanish with a grandiose imperial tone ("crónica colonial absurda").

### Deployment

`vercel.json` has a catch-all rewrite to `index.html` for SPA support. The app is deployed from `main` to Vercel. Connect the GitHub repo via the Vercel dashboard to trigger auto-deploys on push to `main`.

Preview deployments (from `staging` pushes) are protected by Vercel login by default. To test on a phone without logging in, either disable Deployment Protection in the Vercel dashboard (Project → Settings → Deployment Protection) or use `npm run dev -- --host` locally.
